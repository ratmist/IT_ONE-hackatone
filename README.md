# Платформа обнаружения подозрительных операций

Сервис для автоматизированного анализа транзакций, выявления подозрительных операций  и управления правилами обнаружения мошенничества в режиме реального времени. Проект ориентирован на снижение операционных рисков, повышение скорости обработки  и улучшение наблюдаемости за транзакционными потоками.



## Проблематика

Банковские и платёжные системы ежедневно обрабатывают **тысячи транзакций**,  
часть из которых может быть мошеннической.

Основные проблемы:
- жёстко зашитая логика проверки операций  
- отсутствие наглядной административной панели  
- сложность добавления и изменения правил  
- зависимость от ручной проверки и человеческого фактора  

Это приводит к снижению надёжности и росту операционных затрат.



## Решение

Единая платформа с **модульной архитектурой**, которая позволяет:

- гибко настраивать правила обнаружения
- просматривать и изменять их через административную панель
- анализировать транзакции в реальном времени
- вести журнал всех операций
- получать уведомления о подозрительных событиях



## Архитектура

Платформа построена по **микросервисному принципу** и разворачивается с помощью `docker-compose`.

## Автоматизация и запуск

Проект полностью автоматизирован и запускается локально:

```bash
git clone <repository-url>
cd <project-directory>
docker-compose up -d --build
```
## Эффективность

По результатам тестирования платформа демонстрирует следующие показатели:

- **Средняя скорость обработки:**  
  ~ 2–3 тыс. транзакций в секунду

- **Пропускная способность:**  
  ~ 120–180 тыс. транзакций в минуту

- **Время выполнения одного правила:**  
  от 1 до 5 мс



## Экономический эффект

| Показатель | До внедрения | После внедрения | Эффект |
|---------|-------------|----------------|--------|
| Время обработки | 50–100 мс | 1–5 мс | Сокращение |
| Ошибки при ручной проверке | 15–20% | 2–5% | Сокращение |
| Обнаружение мошенничества | 70–80% | 95–98% | Увеличение |
| Операционные затраты | 100% | ~35% | Сокращение |

Основной эффект достигается за счёт **автоматизации анализа транзакций**  
и **снижения влияния человеческого фактора**.

## Наблюдаемость

Платформа обеспечивает полный цикл observability:

- сбор статистики транзакций
- аналитика в административной панели
- журнал операций и событий
- управление транзакциями и правилами
- визуализация метрик и логов


## Управление правилами

Реализованы следующие возможности:

- создание и редактирование правил обнаружения
- тестирование правил с выводом результатов
- включение и отключение правил
- просмотр детального описания условий
- анализ результатов срабатывания


## Безопасность

В проекте реализованы базовые меры безопасности:

- использование `.gitignore` для исключения временных и чувствительных файлов
- применение `.env` для конфигурации и параметров окружения
- вынос секретов и чувствительных данных из `docker-compose.yml`
- изоляция сервисов внутри Docker-сети
